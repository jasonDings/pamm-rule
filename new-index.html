1角色介紹

Master Signal
提供者／Master Signal。真正下單交易的人，所有真實倉位與保證金都在他名下的 master account（主帳戶）。
Investor
投資人。把錢放進 pool 跟著賺賠，但在 broker 端沒有自己的獨立倉位，他擁有的是「pool 的一個持份（share）」。
Pool
所有人的錢集中在一個 master account，當作一筆資金一起交易。
Tradingcup平台
負責記帳、估值、結算與出入金，並把 pool 的倉位按持份比例顯示給每個投資人看。平台只記帳與計算，不自己下單交易。
一句話：真錢與真倉位只有一份（在 master account）；投資人手上的是「pool 的持份 %」與對應的結餘（balance），畫面上的「倉位」是按他的持份比例算給他看的示意值。

2核心設計

為什麼用「持份 + 每期分攤損益」？

難題：很多投資人在不同時間進出、每個人佔的比例都不同（可能很小），pool 的價值又隨未平倉部位每秒在變。要怎麼公平記錄「誰擁有多少、誰該分到哪段漲跌」？

先試最直覺的：給每個人真的 lots（倉位）

舉例：pool 持有 US100 共 0.8 lots。投資人 D 佔 0.5%，他「該有」的就是 0.8 × 0.5% = 0.004 lots，已經小於 broker 的最低下單量 0.01 lots，沒辦法真的開成一筆單。

→ 也就是說：就算幫每個人換算出 lots，那也不是真實倉位，比較像「示意用的虛擬倉位」（broker 端其實只有 Master Signal 名下那一整筆）。所以我們不發真倉位給投資人，而是用一個內部帳本，記錄每個人「佔 pool 多少比例」。

做法：記持份 %，每個結算點把當期損益按比例分攤

核心兩個數字：每位投資人有 ①持份 %（share）＝他的結餘 ÷ pool 總 equity；②結餘（balance）＝他這份現在值多少。

每到一個結算點（rollover），系統把該期間 pool 的損益按各人持份比例分攤進每個人的結餘；沒有「換成單位、再換回現金」的轉換動作，就是「你佔 5%，這段賺/賠的 5% 記到你帳上」。

舉例：pool 現值 $200,000，D 放 $1,000 → D 持份 = 1,000 ÷ 200,000 = 0.5%、結餘 $1,000。之後這小時 pool 賺 $4,000 → D 分到 4,000 × 0.5% = +$20，結餘變 $1,020。D 不需要任何真實倉位。

為什麼這樣可行、給真 lots 不行

  • 可無限細分：再小的持份都能用 % 精確表示，不像 lots 卡在最小 0.01。
    例：D 的 0.5% 直接記 0.5% 就精準了；換成 lots 是 0.004 手，broker 連單都收不了。
  • 進場時間公平：你只分攤「進場後」每一期的損益，不會分到進場前的盈虧。
    例：B 在 pool 漲過一波後才進場，那段（A 賺的）已在 A 的結餘裡，B 從進場那一刻起才開始按持份分配。
  • 進出不影響別人：分攤永遠按「當下持份比例」，別人的結餘不因你進出而改變。
    例：D 全額出金（領走自己的結餘）後，其餘人的持份重新正規化，結餘一毛沒少。
  • 每期都清楚：每個結算點都明確記「這期 pool 賺賠多少、各人分到多少」，逐期可追。
    例：報表上能看到「本小時 pool +2%、你的結餘 +$20」。

3名詞與公式

Pool Equity (NAV)
整個 pool 現在值多少 = Balance + Floating P&L，就是 broker 主帳戶的 Equity。
持份 Share %
投資人佔 pool 的比例 = 他的結餘 ÷ pool equity。出入金時重算，平時不變。
結餘 Investor Balance
投資人這一份現在值多少 = pool equity × 他的持份。每期按持份分攤損益後更新。
Floating P&L
未平倉部位的帳面盈虧，隨報價即時變動。
區間損益 Interval P&L
兩個結算點之間，pool 因交易產生的損益（排除出入金）；按持份分攤給投資人。
rollover（每小時結算點）
系統每個整點的估值/結算時點（跟著交易時段）。只認這一刻的 pool equity 為官方淨值，並在此分攤損益、處理出入金。
HWM (High-Water Mark)
投資人結餘（已排除出入金）曾到過的最高點。只有結餘超過這條線才分潤，跌下去再漲回來不重複分潤。
Forward Pricing
出入金用「下一個 rollover」的 pool equity 結算，不是按按鈕當下——防止挑價套利。
Performance Fee
Master Signal 的分潤，只對結餘超過 HWM 的獲利收取（例 20%），於出金時與每週/每月結算。
Margin / Free Margin
Margin＝倉位佔用的保證金；Free Margin＝Equity − Margin，是能開新倉或出金的緩衝。
MS 平 order
出金時 Master Signal 平掉足量倉位變現，釋放對應 margin，不動到別人的部位。
Negative Balance Protection
行情跳空跳過頭時，投資人最慘賠光本金、不會倒欠（結餘下限為 0）。
Min Lot
broker 能接單的最小倉位（外匯常見 0.01，步進 0.01）。比這小的倉位無法在 broker 真實存在。

核心公式（含舉例推演）

分四步看：① 估值算 pool equity → ② D 入金取得持份 →（每小時）③ 按持份分攤當期損益 → ④ D 出金並結算 performance fee。
框內配色：藍＝步驟標題、綠＝代入的數字例子，白＝公式。

【① 估值（每個整點）】 Pool Equity (NAV) = Balance + Floating P&L ← broker 主帳戶的 Equity，例 $200,000 【② 入金（取得持份，與漲跌無關）】 投資人持份 Share = 投資人結餘 / Pool Equity 例 1,000 / 200,000 = 0.5% 投資人結餘 Balance = 入金金額 例 $1,000 【③ 每小時 rollover：按持份分攤當期損益】 區間損益 = 本期 Pool Equity − 上期（排除出入金） 例 這小時 pool +$4,000 投資人當期損益 = 區間損益 × 投資人持份 例 4,000 × 0.5% = +$20 更新結餘 = 舊結餘 + 投資人當期損益 例 1,000 + 20 = $1,020 【④ 出金並結算 performance fee】 出金 Cash(gross) = 要贖回的結餘 例 結餘漲到 $1,080 全領 Performance Fee = max(0, 結餘 − HWM) × Fee Rate 例 (1,080 − 1,000[D 的 HWM]) × 20% = $16 出金 Cash(net) = Cash(gross) − Performance Fee 例 1,080 − 16 = $1,064 【虛擬倉位（UI 投影，與帳務無關）】 虛擬手數 = Pool Lots(symbol) × 投資人持份 例 0.8 × 0.5% = 0.004 lots 虛擬盈虧 = Pool Floating P&L(symbol) × 投資人持份
注意：估值一定要用含 Floating P&L 的 Equity，不能只用 Balance。否則新進場的人會不公平地分到（或被稀釋掉）還沒平倉的盈虧。

4核心流程

五條流程串起整個 Phase 1：入金 → 每小時估值/分攤 → 分潤 → 出金 → 風控/對賬。

完整生命週期（先看全貌）

下圖把投資人的一筆資金，從進場到退場走一遍。先看這六步在做什麼，再對照下方流程圖：

  • ① 入金 Deposit：投資人把錢放進 Master Signal 的 pool（單一主帳戶），錢進到 pool 裡一起被操作。
  • ② 取得持份 Share：依當下 pool equity 算出投資人的持份 %（＝入金 ÷ pool equity）與結餘，記在內部 ledger 上。
  • ③ 交易 + 每小時 rollover：Master Signal 操盤，系統每個整點以 broker equity 結算，把當期損益按持份分攤進每個人的結餘；持份不變、結餘隨損益增減。
  • ④ 分潤 Perf. Fee：只有結餘超過該投資人的 HWM（前高）才收 performance fee，於出金時與每週/每月結算；虧損不收、回本期間不重複收。
  • ⑤ 出金 Redeem：投資人申請贖回結餘；系統在下一個整點通知 Master Signal 平掉足量 order，把 pool 釋放出對應現金。
  • ⑥ 付現金：以該整點的官方 pool equity 結算，付款給投資人並扣減其結餘與持份，這筆資金正式退場。
動作步驟 判斷 結果

① 入金（取得持份）

  • Forward pricing（遠期定價）：D 在 14:00 按下入金 $1,000，但成交要等下一個整點 rollover。系統用該整點的官方 pool equity 算持份，不是用 14:00 的瞬時值，防挑價；等待最多約 1 小時。
  • 算持份：持份 = 入金 ÷ pool equity。例：$1,000 ÷ $200,000 = 0.5%，結餘 $1,000。
  • 公平點：D 從進場那一刻起才開始按持份分攤損益，不會分到他進場前 pool 賺/賠的部分。

② 每小時估值 + 分攤損益

  • 估值來源：每個整點直接以 broker 主帳戶 equity 為準。例：本期 equity 從 $200,000 → $204,000，區間損益 +$4,000，按各人持份分攤進結餘（D 0.5% → +$20）。
  • 每小時 rollover 的取捨：把估值/分攤/出入金頻率從「每天一次」改成「每個整點」，投資人等待從最多約 24 小時縮到約 1 小時；但市場休市時段（週末、收盤）沒有報價，該時段 rollover 會跳過或延到開盤，且 broker 的 swap/利息一天只收一次，會集中落在某個整點，對賬時要容忍這個跳動。
  • 對賬（白話）：拿「系統 ledger 算的總值」對「broker equity」。例：系統算 $204,000、broker 顯示 $203,940（差 $60，多半是 swap/commission），差 0.03% 在容差內。若差距超出容差、兩邊對不上，系統會發出告警，並可暫停出入金、轉人工查核，確認無誤後才繼續。

③ performance fee 結算（HWM）

  • 只對超過 HWM 的部分分潤：A 結餘從成本 $10,000 漲到 $14,400、HWM = $10,000 → performance fee =（14,400 − 10,000）× 20% = $880。
  • 虧損不分潤：若結餘 $9,000 < HWM $10,000 → fee = $0，Master Signal 只在創新高時才有收入。
  • HWM 不重複分潤：A 若上次在結餘 $14,400 結算過（HWM 更新為 $14,400），之後跌到 $11,000 再漲回 $13,000 → 不分潤（還沒過 $14,400）；漲破 $14,400 才再分潤。
  • 結算時點：performance fee 於出金時與每週/每月結算（crystallize），撥入 Master Signal 的 Wallet account。

④ 出金（贖回結餘 + 平倉釋放現金）

  • 真正保護其他人的是「按持份結算」：D 贖回結餘 $1,000；出金後其餘人的結餘不變、持份重新正規化（相對比例不變）。不管 Master Signal 平掉哪一筆 order，剩下的人結餘都不被稀釋。
  • 為何不影響 margin：付 D 的 $1,000 會讓 equity 少 $1,000；而為了生出這 $1,000，Master Signal 平掉某些 order，同時釋放那些 order 佔用的 margin。equity 與 used margin 一起下降，所以 D 出金不會把其他人的 margin 額度抽乾——D 帶走的只是「自己那份結餘」。
  • 系統不需要按虛擬倉位逐筆平倉：它只要算出「需釋放多少現金 = D 的結餘」，平哪些 order 由 Master Signal 自行決定。
  • 虧損中也能出：若 D 結餘跌到 $880 → 領 $880（自負 $120），其他人結餘不變、且無 performance fee。
  • 每小時消化前一小時的申請：出金申請排到下一個整點處理，等待最多約 1 小時。
出金風險與限制（Phase 1）：
① 平倉後仍可能爆倉：Master Signal 平掉部分 order 釋放現金後，剩餘倉位若在 rollover 前大跌，pool 仍可能觸發強平；D 在現金實際付出前仍是持有人，須一起承擔。
② 無自動平倉機制：Phase 1 完全依賴 Master Signal 配合。規範上必須明訂：investor 提出出金後，Master Signal 須在收到通知後、於該 rollover 前完成平倉釋放現金；未配合則出金順延，並應訂違約/罰則條款。

⑤ 風控與強平（broker 原生）

  • 強平不用平台做：margin level = equity ÷ used margin，跌破 50% → broker 自動強平，平台只接收結果。
  • 大家按持份等比例分攤：例：強平讓 pool 虧 $20,000（−10%），則每個人的結餘同步 −10%（D 結餘 $1,000 → $900）。
  • Negative Balance Protection：若跳空使 equity 變負，結餘封底 $0 → D 最多賠光本金、不會倒欠。

6Phase 1 規格

Phase 1 的完整定義：規則、已知風險，以及要開發的功能清單。

A. 帳戶與簽約

  • Master Signal 註冊
    • 綁定一個 master account，所有人的錢集中在這個 broker 主帳戶。
    • 簽一份 LPOA：授權 MS 操盤、約定 performance fee 費率與 HWM，但不能挪用、也不能代為出金。
    • 建立 Wallet account（收益帳戶，與 CopyTrading 共用）——分潤撥入此帳戶。
  • Investor 簽約
    • 服務條款 + 風險揭露。
    • 入金前試算：輸入金額 → 以當前 pool equity 預估持份 % 與結餘。
  • 解除簽約
    • Investor：全額贖回後解除 LPOA 與服務條款。
    • Master Signal：pool 清算與投資人退出流程（與待討論連動）。

B. 每小時 rollover：估值 + 按持份分攤損益

  • 每個整點讀 MT4/MT5 的 equity（= pool 淨值），算出當期區間損益，按各投資人持份分攤進結餘，更新 ledger。
  • 與 broker equity 對賬：確認平台帳上總額（Σ 各人結餘）= broker 帳戶實際 equity（確保小數位一致）。
  • 結算點頻率：跟著交易時段排程；市場休市時段的 rollover 跳過或延到開盤，broker swap 一天一次的跳動於對賬時容忍。

C. 分潤

  • performance fee 依 LPOA（例 20%），只對結餘超過 HWM 的部分。
  • HWM 追蹤（以結餘計，排除出入金）；出金時 + 每週/每月結算（crystallize），分潤撥入 MS 的 Wallet account（MS 收益帳戶，可提領）。

D. Investor 後台

  • 總覽：my 持份 %、結餘、損益。
  • 虛擬持倉：把 Master Signal 的 order 按持份投影成虛擬 lots（只讀）。
  • 入金
    • 選擇要入金的 live account。
    • 依下一個整點的 pool equity（下一個 rollover）算持份並記入結餘，完成入金。
  • 出金
    • 選擇要出金的 live account。
    • 通知 Master Signal，於該 rollover 前平倉釋放現金。

已知風險

① 靠人工平倉的執行風險
出金要 Master Signal 自己及時平倉，他不在線、拖延、或故意不平就會卡住。Phase 1 沒有機器強制執行。
② 跳空/滑價的失效邊界
保證金快見底又碰上跳空時，「按比例平倉自償」會失效；負餘額保護只保證不倒欠，剩下的虧損與滑價算誰要先講清楚。
③ 投資人體驗：出金仍非「秒到帳」，要等下一個整點 rollover（最多約 1 小時，比每日版好很多）；習慣即時到帳的人仍需期望管理。
④ Master Signal 沒有自己的錢在裡面
他虧的不是自己的錢，容易亂冒險、賭翻本。Phase 1 綁住他的只剩「performance fee」，少了「賠錢他自己也痛」這層保險，道德風險高。
⑤ 資金混同的合規風險
大家的錢全在一個帳戶，很多地方法律上可能算「集合投資／要牌照」。「現在沒人管」不等於合法，上線前一定要找律師看。
⑥ 沒有現金緩衝
每筆出金（連小額）都要平倉才有錢付，交易成本與操作負擔變高；留不留現金全看 Master Signal 自律。
⑦ 每小時結算的營運負擔
估值、分攤、對賬從一天一次變成每個整點，計算量與對賬頻率大增；休市時段、swap 一天一次的跳動都要特別處理，瞬時不一致的機率變高。
⑧ 結算時間差：整點定了價但現金晚點付，中間行情再變的差額歸屬要寫死。
⑨ 負餘額保護誰買單：真穿倉變負時那個洞由 broker／平台／Master Signal 誰補，要事先講好。
