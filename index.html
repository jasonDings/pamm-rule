一個資金池、按 Share 分潤 · 每小時 Rollover 結算 · 全程以 Equity 估值

1・一句話 2・名詞速查 3・核心關係 4・出入金 5・Rollover 6・分潤

一句話說明

把所有投資人的錢合成一個資金池（Pool）交給 Master Signal 操盤。投資人不在 broker 開自己的單，而是持有這個池子的一個比例（Share）；他那一份現在值多少，就是他的 個人 Equity。池子每小時結算一次（Rollover），在那一刻把這一小時的賺賠按比例分給大家、處理出入金、結算分潤。

名詞速查（Key Terms）

Pool Equity資金池總權益
整個池現在值多少（broker 主帳戶權益，含未平倉的 Floating P&L）＝ 全部投資人 Equity 加總。
個人 Equity投資人權益
某位投資人「那一份」現在值多少（含他應分到的浮動損益，隨 Rollover 變動）。
Share持份比例
某位投資人佔整個池的百分比 ＝ 個人 Equity ÷ Pool Equity。
成本基礎Cost Basis
投資人投入的本金（淨入金；入金時增加、出金時按比例減少）。
累計獲利Profit
個人 Equity − 成本基礎，也就是他目前帳上賺了多少。
HWMHigh-Water Mark
已被收過分潤的「累計獲利」最高點，只升不降；分潤只對超過它的新獲利收，虧損／回吐不收。改用「獲利」衡量，不是 equity 水位。
Floating P&L浮動損益
未平倉部位的帳面賺賠，隨報價即時變動。
Performance Fee分潤
付給 Master Signal 的績效獎金（例：新獲利的 20%）。
Rollover每小時結算點
系統唯一的結算時刻：分攤損益、處理出入金、結算分潤。

Equity 與 Share（同一件事的兩種看法）

全系統都以 Equity（mark-to-market、含浮動損益） 估值，不用「只算已實現」的 Balance。每個人的權益有兩種表達方式：個人 Equity（那一份值多少錢）和 Share（佔整個池子的比例）。帳上逐筆記錄每個人的 Equity、每個 Rollover 更新；Share 由 Equity 換算而來，用在 Rollover 時「按比例分攤損益」。所以下面兩條不是循環，而是同一筆權益的「比例面」與「金額面」。
Share = 個人 Equity ÷ Pool Equity ← 比例面：他佔池子幾 % 個人 Equity = Pool Equity × Share ← 金額面：那一份值多少錢
其中 Pool Equity ＝ broker 主帳戶權益（含浮動損益）＝ 全部投資人 Equity 加總；每個 Rollover 結算後兩者必然相等（可用於對賬）。
範例A 的 Equity $5,500、Pool Equity $10,800 → Share = 5,500 ÷ 10,800 = 50.93%；反推驗證：$10,800 × 50.93% ≈ $5,500，兩種看法一致。

出入金（Deposit & Withdrawal）

入金 Deposit

投資人提出入金，Master Signal 核准。核准後資金先放著、等下一個 Rollover 才真正進池並換成 Share；在那之前不參與任何賺賠（避免人還沒進場、就分到別人的損益）。
（在 Rollover 入金時執行） 新 個人 Equity = 舊 個人 Equity + Deposit 新 Pool Equity = 舊 Pool Equity + Deposit 新 成本基礎 = 舊 成本基礎 + Deposit ← 入金算本金、不算獲利（HWM不變）
範例C 在 Pool Equity $8,800 時入金 $2,000 → 個人 Equity = $2,000、Pool Equity = 8,800 + 2,000 = $10,800、成本基礎 = $2,000、HWM = $0、Share ≈ 2,000 ÷ 10,800 = 18.52%。

出金 Withdrawal

投資人提出出金，Master Signal 核准，於下一個 Rollover 生效。生效時：先把這一小時的損益分給他、再結算分潤、最後付款。Master Signal 須在該 Rollover 前先平倉、備好現金。
累計獲利 = 個人 Equity − 成本基礎 Performance Fee = max(0, 累計獲利 − HWM) × 出金比例 × Fee Rate （全額出金時比例 = 100%） 實付金額 = 出金金額 − Performance Fee 新 Pool Equity = 舊 Pool Equity − 出金金額 ← Performance Fee 撥入 MS Wallet
範例B 成本基礎 $2,000、Equity $2,200、HWM $0 → 累計獲利 = 2,200 − 2,000 = $200 → Performance Fee = (200 − 0) × 100% × 20% = $40，實付 = 2,200 − 40 = $2,160。

每小時 Rollover 流程（固定順序，不可調換）

以下四步以同一情境貫穿：起始 Pool Equity $10,000（Master 50% = $5,000、A 30% = $3,000、B 20% = $2,000），本期交易賺 +$1,000，B 申請全額出金、C 申請入金 $2,000（均已核准），Fee Rate 20%。
1

分攤本期損益（Allocate P&L，用舊 Share）

用「上一個 Rollover 的 Share」把這一小時的交易損益，按比例分進每個人的 Equity。
本期損益 = 本期 Pool Equity − 上期 Pool Equity 個人分得 = 本期損益 × 舊 Share 新 個人 Equity = 舊 個人 Equity + 個人分得
範例Pool Equity 10,000 → 11,000，本期損益 = $1,000；Master 1,000 × 50% = +$500 → Equity $5,500（A +$300 → $3,300、B +$200 → $2,200）。
2

處理出金（Process Withdrawals）

逐筆結算分潤、付款、扣減 Equity，全額者退場。分潤只收「超過HWM的新獲利」。
Performance Fee = max(0, 累計獲利 − HWM) × 出金比例 × Fee Rate 新 個人 Equity = 舊 個人 Equity − 出金金額 新 Pool Equity = 舊 Pool Equity − 出金金額
範例B 成本基礎 $2,000、Equity $2,200、HWM $0 → 累計獲利 $200 → Performance Fee = 200 × 20% = $40，實付 $2,160；Pool Equity 11,000 → $8,800，B 退場。
3

處理入金（Process Deposits）

把待入金的資金正式進池、分配 Share、把入金加進成本基礎。
新 個人 Equity = 舊 個人 Equity + Deposit 新 Pool Equity = 舊 Pool Equity + Deposit 新 成本基礎 = 舊 成本基礎 + Deposit （HWM不變）
範例C 入金 $2,000 → Equity $2,000、Pool Equity 8,800 → $10,800、成本基礎 $2,000、HWM $0。
4

重算 Share（Recompute Shares）

用更新後的個人 Equity，重算每個人的 Share，供下一小時使用。
個人 Share = 個人 Equity ÷ Pool Equity 對賬驗證：全部投資人 Equity 加總 = Pool Equity
範例全部 Equity 加總 = 5,500 + 3,300 + 2,000 = $10,800 = Pool Equity ✓；Share → Master 50.93%、A 30.56%、C 18.52%。

分潤（Performance Fee）

原則一句話：只對「還沒被收過分潤的新獲利」抽成。每位投資人記兩個數——成本基礎（本金）與 HWM（已被收過分潤的獲利高水位，只升不降）。先算累計獲利，再對「超過HWM」的部分收；虧損或回吐不收、回本期間不重複收。結算時點：出金時 ＋ 每週／每月。部分出金採 pro-rata：按出金比例對新獲利收費，並把成本基礎與HWM同比例縮小。
累計獲利 = 個人 Equity − 成本基礎 Performance Fee = max(0, 累計獲利 − HWM) × Fee Rate ← 只收「新獲利」 收費後 → HWM = max(HWM, 累計獲利) ← 高水位只升不降
範例 · 收費成本基礎 $10,000、Equity $12,100、HWM $0 → 累計獲利 $2,100 → Performance Fee = (2,100 − 0) × 20% = $420，實拿 $11,680；HWM更新為 $2,100。
範例 · 回吐不重複收（同一投資人續抱）回檔到 $11,000 不收；之後再漲到 $13,000 → 累計獲利 $3,000、HWM $2,100 → 只對新增的 (3,000 − 2,100) = $900 收 = $180。
範例 · 部分出金成本基礎 $10,000、Equity $12,000、HWM $0，領一半 $6,000 → Performance Fee = (12,000 − 10,000 − 0) × (6,000 ÷ 12,000) × 20% = $200；之後成本基礎 → $5,000、HWM → $0（均 ×½）。
